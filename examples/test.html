air-monitor

A live walk-through of the Monitor chaining API

The air-monitor package provides a compact data model and a Monitor class for working with hourly air quality data (PM2.5). A monitor object holds two aligned Arquero tables — meta (one row per device deployment) and data (an hourly, UTC time series) — linked by deviceDeploymentID.

This page loads live AirNow data from the USFS AirFire archive (the most recent ~10 days, updated every few minutes), then filters, summarizes, and exports it. Everything is loaded from CDN with no build step. Loading real data over the network takes a few seconds — watch the status banner below.

Loading & filtering

new Monitor() creates an empty monitor; await monitor.loadLatest("airnow") populates it from the archive. filterByValue(column, value) returns a new monitor restricted to metadata rows where meta[column] === value, and count() reports how many time series remain.

A single deployment

getIDs() lists the deviceDeploymentIDs in a monitor. Given one ID, getMetaObject(id) returns that deployment's full metadata row, and getTimezone(id) returns its Olson timezone.

Time series: raw PM2.5 & NowCast

getPM25(id) returns the raw hourly PM2.5 values, and getNowcast(id) returns the NowCast — the EPA's weighted-average smoothing used for real-time AQI reporting (computed by the companion air-monitor-algorithms package). Missing hours are represented as null.

Daily statistics

getDailyStats(id) trims the series to full local-time days (DST-aware) and returns { datetime, count, min, mean, max } — one entry per local calendar day.

Diurnal (hour-of-day) statistics

getDiurnalStats(id, dayCount) averages each hour of the day across the most recent dayCount days, returning { hour, count, min, mean, max } — useful for seeing the typical daily pollution cycle.

Collapsing many series into one

collapse(deviceID, FUN) aggregates every time series in a monitor into a single series using an Arquero aggregation function ("mean", "max", …). The result is placed at the mean longitude/latitude of the inputs — handy for a regional average.

Current status & GeoJSON export

getCurrentStatus() augments meta with each site's most recent valid timestamp and value. createGeoJSON() turns the deployments into a GeoJSON FeatureCollection ready to drop onto an interactive map.
