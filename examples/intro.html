air-monitor – Basic Example

Purpose

The air-monitor package provides a compact data model and transformation utilities for working with hourly air quality monitoring data. It is built on the Arquero table library and uses Luxon for UTC-anchored datetime handling.

A Monitor object holds two aligned Arquero tables — meta (one row per device deployment) and data (a regular hourly time series in UTC, one column per device) — linked by a shared deviceDeploymentID. The package supports loading data directly from the USFS AirFire monitoring archive.

This page loads live AirNow data, filters to Washington state, and computes a statewide daily average using the package's chaining API.

Libraries Used in This Example

  • air-monitor — provides the Monitor class used throughout this example.
  • Arquero — the table engine used internally by air-monitor for metadata and time-series operations.
  • air-monitor-algorithms — provides the NowCast and daily / diurnal statistics algorithms used internally by air-monitor.

All three packages are loaded from jsDelivr using an import map, which lets the script use standard ES6 import syntax with no build step or bundler. The import map is required because air-monitor's ES module build references arquero and air-monitor-algorithms by their bare package names; the map tells the browser where to fetch them. 


<script src="https://cdn.jsdelivr.net/npm/arquero@8.0.3/dist/arquero.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/air-monitor-algorithms@1.4.2/dist/air-monitor-algorithms.umd.js"></script>
<script src="https://cdn.jsdelivr.net/npm/air-monitor@1.2.5/dist/air-monitor.umd.js"></script>

<script type="module">

Loading AirNow Data

  import { Monitor } from 'air-monitor';

  const m = new Monitor();
  await m.loadLatest("airnow");

loadLatest("airnow") fetches the most recent ten days of AirNow data from the USFS AirFire archive, updated every few minutes. new Monitor() creates an empty instance; the awaited call populates its meta and data tables in place. All timestamps are stored as Luxon DateTime objects in UTC.

Filtering to Washington

  const wa = m.filterByValue("stateCode", "WA").dropEmpty();

filterByValue() returns a new Monitor restricted to metadata rows where the named column equals the supplied value. dropEmpty() removes any device series that contain only null observations over the loaded window. Every transform returns a new Monitor, so calls chain naturally.

Daily Statistics for Washington

  const waAvg = wa.collapse("wa_mean", "mean");
  const waID  = waAvg.getIDs()[0];
  const daily = waAvg.getDailyStats(waID);

collapse() aggregates every Washington monitor into a single synthetic series placed at the mean lat/lon of the inputs. The first argument is a base name for the resulting deviceDeploymentID; the second is an Arquero aggregate function name. getDailyStats() returns daily min/mean/max in local calendar time, correctly accounting for daylight saving time.
