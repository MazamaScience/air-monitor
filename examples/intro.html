Introducing air-monitor

A guided walk-through of the Monitor chaining API

air-monitor serves hourly air quality monitoring data through a small, chainable Monitor class backed by two aligned Arquero tables — meta (one row per device deployment) and data (a UTC hourly time series). This page walks through two real examples.

  1. Right now in Washington — pull the latest data and examine the statewide daily average.
  2. The 2025 Garnet Fire — find a wildfire's smoke in a full year of California data.

Every transform returns a new Monitor, so you can chain steps together safely. Section 1 loads automatically; Section 2 requires a button click because the annual archive is a larger download (~10–20 MB).

Setup

Load air-monitor and its two runtime dependencies directly from jsDelivr — no build step, no bundler. Script load order matters: Arquero and the algorithms package must be present before air-monitor is evaluated. 

<!-- 1. Arquero — the table engine used internally by air-monitor -->
<script src="https://cdn.jsdelivr.net/npm/arquero@7.2.1/dist/arquero.min.js"></script>

<!-- 2. NowCast and daily / diurnal statistics -->
<script src="https://cdn.jsdelivr.net/npm/air-monitor-algorithms@1.4.2/dist/air-monitor-algorithms.umd.js"></script>

<!-- 3. air-monitor itself -->
<script src="https://cdn.jsdelivr.net/npm/air-monitor@1.2.6/dist/air-monitor.umd.js"></script>

The UMD build wraps exports in a namespace object. Unwrap it once before calling new Monitor(): 

const Monitor = window.Monitor.Monitor ?? window.Monitor.default ?? window.Monitor;

1. What's in the air right now? (Washington)

loadLatest() fetches roughly the last ten days from the AirNow archive — thousands of monitors across the country, updated every few minutes. new Monitor() creates an empty monitor; the await call populates it in place. 

const m = new Monitor();
await m.loadLatest("airnow");

console.log(`Loaded ${m.count()} device deployments`);
console.log(`Time range: ${firstHour(m)} → ${lastHour(m)}`);
Loading…

Narrow to Washington with filterByValue(), then drop monitors that reported nothing with dropEmpty(). Any metadata column can be used as a filter key — string or numeric. 

const wa = m.filterByValue("stateCode", "WA").dropEmpty();

console.log(`${wa.count()} Washington monitors reporting`);
Waiting…

collapse() averages all those monitors into a single synthetic series placed at the mean lat/lon of the inputs. getDailyStats() then returns daily min/mean/max in local calendar time — DST-aware, no extra configuration needed. 

// All WA monitors → one state-average series
const waAvg = wa.collapse("wa_mean", "mean");
const waID  = waAvg.getIDs()[0];

// Daily statistics in local calendar time (DST-aware)
const daily = waAvg.getDailyStats(waID);
// Returns { datetime, count, min, mean, max } — one entry per local day
Waiting…

Because this is live data your numbers will differ from any shown here. On a calm summer day Washington often reads in the 1–5 µg/m³ range; a sustained climb into the tens or higher usually means wildfire smoke.

2. Finding a wildfire's smoke (the 2025 Garnet Fire)

The Garnet Fire was started by lightning on August 24, 2025 in the Sierra National Forest, south of Yosemite. On September 7 it grew by roughly 10,000 acres in a single day — threatening the giant sequoias of McKinley Grove — and pushed dense smoke northeast into Yosemite's high country and the Mono Lake basin.

loadAnnual() gives us a whole year of data so we can look back and watch that smoke arrive. The archive for a full year is larger than the latest or daily files — click below when ready. 

const year = new Monitor();
await year.loadAnnual(2025);

console.log(`${year.count()} device deployments in the 2025 archive`);
Click the button above to start.

There is no built-in geographic filter, but the raw Arquero metadata table is always one attribute away as m.meta. Convert it to plain objects with .objects(), apply a lat/lon bounding box in plain JavaScript, then pass the matching IDs to select(): 

// Filter to California first
const ca = year.filterByValue("stateCode", "CA");

// m.meta is an Arquero table — .objects() converts it to a plain JS array
const areaIDs = ca.meta.objects()
  .filter(r =>
    r.latitude  >= 37.6  && r.latitude  <= 38.3 &&
    r.longitude >= -119.6 && r.longitude <= -118.9
  )
  .map(r => r.deviceDeploymentID);

const area = ca.select(areaIDs).dropEmpty();
console.log(`${area.count()} monitors in the Yosemite / Mono Lake area`);
Waiting…

Zoom into the first half of September with filterDatetime(), collapse the area into one signal, and look at the daily average. The smoke arrives on September 8, the day after the fire's big run: 

const earlySep = area.filterDatetime("2025-09-01", "2025-09-15", "UTC");

// Collapse all area monitors into one mean series and get daily stats
const areaAvg   = earlySep.collapse("area_mean", "mean");
const areaID    = areaAvg.getIDs()[0];
const areaDaily = areaAvg.getDailyStats(areaID);
Waiting…

The area average already jumps well above 200 µg/m³, but averaging hides how extreme it got at individual sites. Which monitor was worst, and when? getPM25() on each monitor lets us find out directly: 

const ids = earlySep.getIDs();
let worstID = ids[0], worstMax = -Infinity;
for (const id of ids) {
  const arr = earlySep.getPM25(id);
  const max = arr.reduce((m, v) => (v != null && v > m ? v : m), -Infinity);
  if (max > worstMax) { worstMax = max; worstID = id; }
}

const worstMeta = earlySep.getMetaObject(worstID);

// Locate the datetime of the peak value
const pm25     = earlySep.getPM25(worstID);
const dtimes   = earlySep.getDatetime();
const peakIdx  = pm25.reduce((best, v, i) =>
  v != null && v > (pm25[best] ?? -Infinity) ? i : best, 0);
const peakTime = dtimes[peakIdx];
Waiting…

Pull that monitor's hourly series with getPM25() and watch the smoke front pass through over a few days: 

// Zoom further to just the critical 5-day window
const fiveDays = earlySep.filterDatetime("2025-09-06", "2025-09-10", "UTC");
const hourly   = fiveDays.getPM25(worstID);
const hours    = fiveDays.getDatetime();
Waiting…

For context, the U.S. air-quality index calls a PM2.5 level above 250 µg/m³ "Hazardous" — the maroon, everyone-stays-indoors category. Lee Vining's hourly reading hit roughly 1,100 µg/m³ on September 8, more than four times that threshold, as the Garnet Fire's smoke poured over the Sierra crest. A day or two later the air had cleared again — a vivid reminder of how sudden and localized wildfire smoke can be, and why per-monitor hourly data matters.

Where to go next

A few methods not used here:

  • getCurrentStatus() — each monitor's latest valid timestamp and value
  • createGeoJSON() — export deployments as a GeoJSON FeatureCollection
  • combine(other) — merge two Monitor instances into one
  • loadDaily("airnow") — rolling 45-day archive, updated daily

The raw tables are always one attribute away: m.meta (one row per monitor, an Arquero table) and m.data (an hourly UTC time series, one column per monitor). See the README for the full public API.
